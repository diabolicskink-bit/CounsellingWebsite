# Anti-Generic Design Reference

Use this reference when a user asks for distinctive work, wants to avoid AI tropes, or when the first concept appears transferable to unrelated organisations.

## Contents

1. What “AI-looking” means
2. Why convergence happens
3. Trope diagnosis
4. Productive-friction workflow
5. Multi-axis critique

## 1. What “AI-looking” means

Do not infer authorship from appearance. Humans use trends, templates, component libraries, and popular fonts; AI can produce distinctive work under strong direction.

Use “AI-looking” only as shorthand for statistically safe decisions with weak contextual justification. The issue is convergence, not a forbidden ingredient.

Run the transferability test: replace the logo, accent colour, and nouns with a competitor's. If the design remains fully plausible, it encodes the category more than the subject.

## 2. Why convergence happens

- Broad prompts activate broad priors.
- Trend galleries and training/reference distributions already contain sameness.
- Component libraries make a few structures disproportionately cheap.
- Generated first drafts appear complete and suppress exploration.
- Reviewers select polish before examining content fit or hierarchy.
- Broad regeneration loses good local decisions.
- Speed is mistaken for resolution.
- One generated page scales into a whole-site template.

Intervene at every stage:

| Stage | Risk | Intervention |
| --- | --- | --- |
| Brief | “Clean, modern, premium” | Specify audience state, content, tension, thesis, and exclusions |
| Reference | Only competitors and current web galleries | Add books, objects, places, photography, culture, and historical design |
| Generation | Unstated decisions become common priors | Expose assumptions and create structurally opposed directions |
| Selection | The most finished draft wins | Review unbranded concepts across several dimensions |
| Refinement | One cliché becomes another | Edit exact relationships locally and preserve resolved choices |
| Production | Library availability determines the page | Adapt components to content and add page-level composition |
| Review | “Looks good” hides causes | Critique hierarchy, layout, type, colour, imagery, states, and fit separately |
| Scale | One composition repeats everywhere | Define a grammar with controlled content-shaped variation |

## 3. Trope diagnosis

Treat these as questions, not bans:

| Pattern | Ask |
| --- | --- |
| Universal centred hero | Does this hierarchy suit the visitor and content, or merely the generator? |
| Purple-blue aurora or warm beige editorial styling | Is the palette owned by the subject and art direction? |
| Bento grid or card soup | Are these genuinely comparable or independent objects? |
| Large rounded translucent surfaces | Does the material explain layering, state, or interaction? |
| Fashionable default type | Do the metrics, voice, language, and setting belong here? |
| Thin-line icon in every heading | Does the symbol improve recognition or merely decorate? |
| Ambient blobs, mesh, grain, dot grids, or chrome | Does the atmosphere arise from the concept? |
| Repeated fade-and-rise reveals | Does motion explain sequence, state, or causality? |
| Logo cloud or testimonial ticker | Is the proof verifiable, contextual, and accessible? |
| Generic AI or stock imagery | Does the image carry evidence, explanation, narrative, or deliberate atmosphere? |
| Vague “unlock/elevate/reimagine” copy | Can the claim be concrete and bounded? |
| Large unsupported metrics | Is scope, source, date, and relevance visible? |
| Perfectly alternating split sections | Is the rhythm shaped by content or a template? |

The strongest warnings are accumulation and transferability. Any single ingredient may be correct.

## 4. Productive-friction workflow

### Write the decision brief

Name audience, page job, content, evidence, trust, constraints, character tensions, conventions to preserve, and conventions to challenge.

### Audit the category

Record common hero shapes, palettes, type categories, imagery, proof patterns, interaction, and section sequences. Decide which conventions support recognition and which cause indistinguishability.

### Broaden the reference field

Use references for explicit properties rather than wholesale imitation. Label what to borrow: crop, grid tension, palette proportion, material, rhythm, caption behaviour, or transition.

### Generate contrastive concepts

Require concepts that differ in structure, density, type, image role, material, and motion. Do not accept palette swaps as alternatives.

### Select a thesis

Choose one sentence that explains why the design looks this way. Preserve strong local decisions during iteration.

### Insert review gates

- expose assumptions before generation;
- use real content before high-fidelity polish;
- compare concepts without logos or brand colours;
- make local edits instead of whole-page regeneration;
- review accessibility and performance before final selection;
- require human sign-off after responsive implementation.

## 5. Multi-axis critique

Discuss dimensions independently:

- brief fidelity;
- information hierarchy;
- macro composition;
- layout and alignment;
- typography;
- colour harmony and role clarity;
- image or illustration direction;
- material coherence;
- interaction and motion;
- responsive behaviour;
- accessibility;
- performance;
- trust and evidential specificity;
- distinctiveness;
- maintainability.

Never rely on one automated aesthetic score. A complete-looking page can remain unresolved on most dimensions.

Do not create an “anti-AI aesthetic.” Negative prompts and banned lists only move the default. Generate from subject, audience, content, strategy, and art direction; diagnose tropes afterward.
