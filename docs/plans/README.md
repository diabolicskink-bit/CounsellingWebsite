# Plans

This folder is for plans of action and thinking documents. Completed plans may remain here as implementation records when their status and any consciously deferred or superseded phases are clearly documented.

- Put planning notes here when you want a proposed approach captured in `.md`.
- Update a retained plan's status when the work is completed, consciously closed, or superseded so historical task lists are not mistaken for active scope.
- Keep filenames descriptive so they are easy to find from the documents page.
- The dev documents page groups everything in this folder under `Plans`.
